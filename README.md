**Алгоритм определения занятости парковки по четырем камерам с перекрытиями**

Данный алгоритм считывает rtsp-видеопотоки с камер и в реальном времени определяет занятые и свободные места на видеопотоке с каждой из 4-х камер, а далее аггрегирует результаты и выводит на вид сверху результат.

Для детекции и сегментации машин на изображении используется предобученная модель YOLOv8nano-seg. Далее для каждой найденной машины определяется занимаемое ей место.

Так как каждое место видно достоточно хорошо как минимум 2-мя камерами, а из-за перспективы алгорим может плохо определять место машин на далеком расстоянии от камеры или при перекрытиях, та логика аггрегации выглядит следующим образом: место считается действительно занятым, если оно определяется как занятое двумя или большим количеством камер.

Запустить проект можно с помощью скрипта start_pipeline.sh, предварительно установив в среду необходимые зависимости из cvpipeline/requirements.txt.
