**Алгоритм определения занятости парковки по четырем камерам с перекрытиями**

Данный алгоритм считывает rtsp-видеопотоки с камер и в реальном времени отрисовывает занятые и свободные места на общем плане парковки(вид сверху).

Для выполнения задания был взят датасет CHAD (https://github.com/TeCSAR-UNCC/CHAD):
- парковка покрыта 4-мя камерами с перекрытием зон
- плоская уличная парковка на 14 машиномест
- Места размечены в соотвествии с планом и пронумерованы.

1. Развертка перспективы и соотношение мест на перспективах
с top view

Отображение перспективы в вид сверху осуществлялся с помощью поиска матрицы гомографии (метод cv2.findHomography) для каждой из 4-ех камер по 4-8 точкам. Алгоритм реализован в файле homography/homography_hand.py


2. Отрисовка объемных областей для каждого места

Из-за отсутствия параметров камер для отрисовки вертикальных линий была использована приблизительная формула:

v = vp - p_bottom

k = C * car_height / d

p_top = p_bottom - v * k

vp - точка схода вертикалей на изображении
p_bottom - точка основания фигуры
d - расстояние от камеры до точки основания (на top view)
C - подбирается (по смыслу содержит внутренние параметры камеры)

Длина вертикали обратно пропорциональна расстоянию до камеры, и лежит на линии соединения bottom и vp точек

Алгоритм отрисовки "обемных мест" можно найти в файле homography/show_places.py

3. Создание алгоритма определения занятого места для одной камеры

В качестве алгоритма детекции машин на каждом кадре использовалась модель
YOLOv8nano-seg. Считается, что машина заняла место, если:

- более 80% машины находится внутри маски объемного места
- более 20% плоского места занято этой машиной

Из всех таких мест берется место с наибольшим значением iou для этой машины
Данный алгоритм выполняется для всех найденных машин

Алгоритм реализован в файле cvpipeline/src/occuped_finder.py

4. Реализация алгоритма поиска занятого места совокупностью камер

Так как каждое место видно достаточно хорошо минимум 2-мя камерами, было
принято решение: считать место занятым, когда на 2-х и более камерах место определено как занятое

Алгоритм реализован в файле cvpipeline/src/pipeline.py

5. Оптимизация алгоритма для работы в real time

Для минимизации задержек обработки real time потока были приняты следующие меры:

- ускорение инференса с помощью gpu
- чтение 4-х видеопотоков отдельными процессами
- пересчет масок машин и занятых мест каждый 30-й кадр
- пересчет масок машин и занятых мест в фоновом потоке, без
прерывания отображения кадров, отображение обновленных занятых
мест по готовности
- расчет масок мест до начала основного пайплайна

6. Создание синхронных отрезков видео для демонстрации в capcut

Так как видео изначально не синхронизированы, пришлось заняться самой неинтересной работой - поиск совпадающих отрывков видео

Итого: 4 синхронизированных отрезка видео с разными ситуациями длительностью
от 1 до 8 минут

7. Симуляция rtsp-потоков данных с .mp4 видео

Для симуляции работы в реальном времени был выстроен следующий
пайплайн обработки синхронизированных видео:

video.mp4
   │
FFmpeg
   │
MediaMTX сервер
   │
CV pipeline

Все 4 потока воспроизводятся параллельно
